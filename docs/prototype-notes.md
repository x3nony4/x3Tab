# Prototype Notes — UI 方案验证

**日期**: 2026-06-23
**原型位置**: `entrypoints/newtab/`（throwaway，正式实施前删除）
**问题**: 新标签页三个布局方案哪个合适？

## 验证结果

**选定**: 方案一 · 标准居中布局

### 布局参数

| 属性           | 值                                        |
| -------------- | ----------------------------------------- |
| 时钟垂直位置   | `padding-top: 18vh`，偏上区域             |
| 时钟字号       | 96px，字重 200                            |
| 搜索框宽/高    | 560px / 52px                              |
| 搜索框间距     | 距离时钟 48px                             |
| Dock 位置      | `position: fixed; bottom: 0`，水平居中    |
| 右上角主题按钮 | `position: fixed; top: 20px; right: 20px` |

### 深色主题配色

| 元素           | 值                                                   |
| -------------- | ---------------------------------------------------- |
| 页面背景       | `#1c1c1e`（中性黑，不偏蓝）                          |
| 主文字         | `#e4e4e4`                                            |
| 次要文字       | `#888` / `#666`                                      |
| 搜索框背景     | `rgba(255,255,255,0.04)`                             |
| 搜索框边框     | `rgba(255,255,255,0.06)`                             |
| 搜索框聚焦边框 | `rgba(255,255,255,0.15)`（白色高亮，非蓝色）         |
| Dock 背景      | `rgba(28,28,30,0.6)` + `backdrop-filter: blur(24px)` |
| 引擎图标       | `rgba(255,255,255,0.08)`，hover 至 `0.14`            |

### 交互约束

- 全局 `user-select: none`，仅 `input/textarea` 可选择文本
- 时钟、Dock 栏文本均不可选中

### 否决的方案

- **方案二（搜索为主）**: 搜索框过大（72px 高），时钟退化为角落元素，偏离"时钟+搜索"并重的定位
- **方案三（时钟为主）**: 时钟 140px 过大，屏幕利用率低，搜索框太紧凑
